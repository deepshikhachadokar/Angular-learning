One-way DataBinding

Imagine you have a whiteboard where you write some information. People can see what you’ve written, but they cannot write back on the board or change it. The information flows in one direction only, from you (the app) to the whiteboard (the screen).

Example:

  • You (the app) write: "Hello, John" on the whiteboard.
  • Everyone sees "Hello, John" on the whiteboard.
  • If you decide to change it to "Hello, Alice," the whiteboard updates automatically.
  • But if someone tries to change "Alice" to "Mark" on the whiteboard, they cannot do it, because they only see the data and can't send it back to you.

    • property Binding /Attribute Binding

    property binding in Angular is like connecting the app's data (or logic) to the HTML element's property. It allows you to dynamically set values for HTML properties based on the app’s logic. Think of it as feeding data from your app to an HTML element's attribute or property.

    How Does It Work?

    In Angular, property binding uses square brackets [] to bind a property of an HTML element to a value from your app’s component.

    • property refers to the HTML element’s property (like src, disabled, value, etc.).
    • expression is the value or logic from the Angular component.

    Event Binding

    Event binding type is when information flows from the view to the component when an event is triggered. The event could be a mouse click or keypress. The view sends the data to update the component.

    Unsurprisingly, it is the exact opposite of property binding, where the data goes from the component to the view.

    • Clicking a button
    • Typing in a text box
    • Hovering over something with a mouse

    Event Binding is how Angular connects these actions (events) to your code so you can decide what happens next

    Click Event

    Let’s say you have a button. When someone clicks the button, you want a message to appear.

    What happens:
    • The user clicks the button.
    • Angular sees the click event and runs the showMessage() method.
    • The method shows an alert box saying, “Hello! You clicked the button.”
