Two-Way Binding

Two-way data binding in Angular allows data to flow in both directions between the view and the component class. This means that users can make changes to the data using the UI, and the data will automatically update the view.
The ngModel directive is commonly used to implement two-way data binding in Angular. This directive binds the values of HTML controls, such as input, select, and textarea, to a variable in the controller's scope.

Two-way binding lets data flow both ways:

  • From the text box to your code.
  • From your code back to the text box.

    • Your name is: {{ username }}

signal

A Signal is like a container that holds a value. When the value changes, any part of the application that depends on the signal is automatically updated.

It’s similar to variables, but reactive—it notifies components and views when the value changes.

Key Features of Signals

  • Reactive:
    Signals automatically update when their value changes.
  • Simple API
    Easy to create, read, and update signals.
  • Efficient:
    Signals work seamlessly with Angular's change detection.
  • Computed Signals:
    You can derive new values reactively based on existing signals.
  • Effect:
    Run side-effects automatically when a signal changes.
