Data Binding

data binding is like a bridge that connects the data in your app with what the user sees on the screen (the UI). It keeps everything in sync without you needing to update things manually.

  • One Way Binding
  • Two Way Binding

One-Way Data Binding

Think of it like writing on a whiteboard. You write something, and others see it. The app updates the screen whenever the data changes, but the user can't change the data directly.

Example:

If you set your name in the app as "John," the screen will show "John."

Two-Way Data Binding

This is like talking to someone over the phone. If you say something, they hear it, and if they reply, you hear it back. Changes on the screen (like user input) update the app's data, and changes in the app's data update the screen.

Example:
  • You type "Alice" in a text box.
  • The app instantly updates and shows "Hello, Alice!" somewhere on the screen.
When you click on "Data Binding" in the navigation menu, Angular routes you to the DataBindingComponent. If you added the fragment (#info), it scrolls directly to the section with that id.
